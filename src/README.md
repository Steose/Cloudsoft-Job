# src folder

This folder contains the application source code for the CloudSoft Job Portal.

## How the projects fit together

1. `Cloudsoft.Core` contains the shared business logic.
   Both the web app and the API use this project for models, services, repositories, and configuration options.

2. `Cloudsoft.Web` contains the MVC website.
   This is the browser-facing app with controllers, Razor views, login/register pages, job listing pages, and static files.

3. `Cloudsoft.Api` contains the HTTP API.
   This is used by external clients or services that want to read or create job postings through JSON endpoints.

4. `Cloudsoft.Web` and `Cloudsoft.Api` both reference `Cloudsoft.Core`.
   This keeps the job and employer logic in one place instead of duplicating it.

5. Tests are kept outside this folder in `test/Cloudsoft.Tests`.
   The `src/` folder should mainly contain production application code.

## `Cloudsoft.Core`

- `Models/` contains the main data objects:
  `JobPosting`, `EmployerUser`, and `EmployerAccount`.

- `Services/` contains business actions:
  `JobPostingService` creates, reads, updates, deletes, and toggles jobs.
  `EmployerAuthenticationService` registers employers and validates login credentials.

- `Services/Interfaces/` contains service contracts used by controllers.

- `Repositories/` contains persistence logic:
  in-memory repositories, resilient fallback repositories, and repository interfaces.

- `Repositories/Interfaces/` defines what repository methods must exist.

- `Data/InMemory/` contains the local in-memory database used for development and fallback storage.

- `Data/MongoDb/` contains MongoDB repository implementations and MongoDB mapping setup.

- `Options/` contains configuration classes for feature flags, MongoDB, and Key Vault.

- `Cloudsoft.Core.csproj` defines the Core project and its dependencies.

## `Cloudsoft.Web`

- `Program.cs` starts and configures the MVC web app.
  It registers services, authentication, repositories, routing, static files, MongoDB fallback behavior, and Key Vault configuration.

- `Controllers/` handles browser requests:
  `AccountController` handles register, login, and logout.
  `JobsController` handles job listing, details, creating jobs, and toggling active status.
  `HomeController` handles simple home/privacy pages.

- `Models/` contains view models used by web forms, such as login and registration.

- `Views/` contains Razor pages shown in the browser.
  `Account/` has login and register pages.
  `Jobs/` has listing, details, and create job pages.
  `Shared/` has the common layout, validation scripts, and error page.

- `wwwroot/` contains public static files such as CSS, JavaScript, favicon, Bootstrap, and jQuery.

- `appsettings.json` and `appsettings.Development.json` contain web app configuration.

- `Properties/launchSettings.json` contains local development launch profiles.

- `Config/`, `Middleware/`, `Repositories/`, and `Services/` are placeholder folders for future web-specific code.

- `WebAssemblyMarker.cs` is a small marker type used by integration tests to start the web app.

- `Cloudsoft.Web.csproj` defines the Web project and references `Cloudsoft.Core`.

## `Cloudsoft.Api`

- `Program.cs` starts and configures the API app.
  It registers controllers, services, repositories, MongoDB fallback behavior, and Key Vault configuration.

- `Controllers/JobsController.cs` exposes job posting API endpoints:
  get all jobs, get active jobs, get one job by ID, and create a job.

- `Controllers/HealthController.cs` exposes a health endpoint for checking that the API is running.

- `appsettings.json` and `appsettings.Development.json` contain API configuration.

- `Properties/launchSettings.json` contains local development launch profiles.

- `ApiAssemblyMarker.cs` is a small marker type used by integration tests to start the API app.

- `Cloudsoft.Api.csproj` defines the API project and references `Cloudsoft.Core`.

## Generated folders

- `bin/` contains compiled build output.
- `obj/` contains temporary build and restore files.

These folders are generated by .NET and should not be edited manually.
