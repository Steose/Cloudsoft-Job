# Cloudsoft.Tests

This folder contains the automated tests for the CloudSoft Job Portal.
The tests use xUnit.

## How the tests are organized

1. `Cloudsoft.Tests.csproj` defines the test project.
   It references xUnit, the ASP.NET Core test host, and the three app projects: `Cloudsoft.Core`, `Cloudsoft.Web`, and `Cloudsoft.Api`.

2. `AssemblyInfo.cs` configures xUnit test behavior.
   It disables parallel test execution so integration tests that change temporary configuration do not interfere with each other.

3. `Unit/` contains tests for individual classes.
   These tests call services, repositories, and controllers directly without starting the full web app.

4. `Integration/` contains tests for real app behavior.
   These tests start the API or web app in memory and send HTTP requests to check complete flows.

## Unit tests

1. `JobPostingRepositoryTests.cs`
   Tests the in-memory job repository CRUD behavior:
   create, read, update, delete, get all jobs, and get active jobs.

2. `JobPostingServiceTests.cs`
   Tests job business logic:
   creating jobs, generating missing IDs, setting created dates, updating, deleting, and toggling active status.

3. `EmployerAuthenticationServiceTests.cs`
   Tests employer registration and login:
   required values, duplicate email prevention, valid login, and invalid login.

4. `WebJobsControllerTests.cs`
   Tests the MVC jobs controller:
   listing jobs, viewing details, showing the create form, validating deadlines, creating jobs, and toggling active status.

5. `ApiJobsControllerTests.cs`
   Tests the API jobs controller:
   get all jobs, get active jobs, get by ID, not found responses, and create responses.

6. `ViewModelValidationTests.cs`
   Tests form validation:
   login fields, registration fields, matching passwords, and required job posting fields.

7. `ControllerTestHelpers.cs`
   Contains helper classes used by controller unit tests, such as fake services and temporary controller setup.

## Integration tests

1. `TestWebApplicationFactories.cs`
   Creates in-memory versions of the Web app and API app for integration testing.

2. `ApiJobEndpointTests.cs`
   Sends HTTP requests to `/api/jobs`.
   It checks creating jobs, reading all jobs, reading active jobs, reading one job, and missing job responses.

3. `WebUserStoryTests.cs`
   Tests browser-style user stories:
   anonymous users can browse jobs, job details work, protected pages redirect to login, employers can register, log in, and log out.

4. `PersistenceConfigurationTests.cs`
   Tests storage configuration:
   the apps use in-memory repositories when MongoDB is disabled and resilient MongoDB repositories when MongoDB is enabled and configured.

5. `AntiforgeryToken.cs`
   Reads anti-forgery tokens from HTML forms so integration tests can submit login, register, and logout forms.

6. `EnvironmentVariableScope.cs`
   Temporarily sets environment variables for configuration tests and restores the old values afterward.

## How to run the tests

From the repository root:

```bash
dotnet test test/Cloudsoft.Tests/Cloudsoft.Tests.csproj
```

Or run the whole solution:

```bash
dotnet test Cloudsoft.sln
```

## Generated folders

- `bin/` contains compiled test output.
- `obj/` contains temporary restore and build files.

These folders are generated by .NET and should not be edited manually.
